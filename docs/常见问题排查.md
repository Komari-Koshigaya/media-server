## 一、连接问题

### 1.1 手机无法访问服务器

**排查步骤**：

```
1. 确认手机和电脑连的是同一个 WiFi / 同一个路由器
2. 确认服务器在运行（控制台有日志输出）
3. 确认访问地址正确（看启动日志中的"手机访问"地址）
```

**最常见原因：手机连错了 WiFi**。电脑有多个网卡时（以太网 + WLAN），启动日志显示的 IP 可能不是手机所连网络的 IP。

查看电脑所有 IP：
```powershell
ipconfig | findstr "IPv4"
```

逐个尝试 `http://<每个IP>:<端口>`。

### 1.2 电脑本地可以访问，手机不行

**原因**：Windows 防火墙拦截了入站连接。

**解决**：以管理员权限运行：
```powershell
netsh advfirewall firewall add rule name="Media Server" dir=in action=allow protocol=tcp localport=18080
```

验证规则已添加：
```powershell
netsh advfirewall firewall show rule name="Media Server"
```

应看到：
```
方向:     入
操作:     允许
协议:     TCP
本地端口: 18080
```

### 1.3 重启电脑后手机又访问不了

**原因**：防火墙规则可能被安全软件清除，或网络配置文件类型变回"公用"。

**排查**：
```powershell
# 检查网络类型
Get-NetConnectionProfile | Select-Object Name, NetworkCategory

# 如果显示 "Public"，改为 "Private"
Set-NetConnectionProfile -Name "你的网络名" -NetworkCategory Private

# 重新添加防火墙规则
netsh advfirewall firewall add rule name="Media Server" dir=in action=allow protocol=tcp localport=18080
```

### 1.4 端口被占用，服务器启动失败

**报错信息**：
```
PermissionError: [Errno 13] error while attempting to bind on address ('0.0.0.0', 8080)
```

**解决**：
```powershell
# 查看占用端口的进程
netstat -ano | findstr :8080

# 杀掉该进程（替换 <PID>）
taskkill /PID <PID> /F

# 或者换一个端口启动
python -m media_server --port 18081
```

## 二、播放问题

### 2.1 视频打不开或一直加载

**可能原因**：
- 视频编码手机浏览器不支持（如 HEVC/H.265、VP9）
- 视频文件损坏

**解决**：页面会自动显示"转码播放"按钮，点击后等待转码完成。大文件转码需要数分钟。

如果转码也失败，点击"下载"按钮下载到手机后用本地播放器打开。

### 2.2 视频播放很慢，等很久才能看

**原因**：视频编码不是 H.264，需要全量重编码。

**优化建议**：
- 优先使用 H.264 编码的 MP4 文件，可以直接复制无需转码
- MKV 文件如果视频轨是 H.264，只转音频（秒级完成）
- 全量重编码的视频会在临时文件缓存 10 分钟，期间再次打开秒开

### 2.3 转码视频暂停后从头开始

**原因**：`-c:v copy` 模式保留源文件的关键帧结构，关键帧稀疏时播放器无法定位到中间位置。

**这是已知限制，非 Bug**。速度优先策略下的取舍：
- 只转音频：秒级完成，但暂停恢复可能从头
- 全量重编码：速度慢，但暂停恢复正常

### 2.4 音频播放没声音或格式不支持

**原因**：浏览器不支持该音频编码（如 FLAC、APE 部分浏览器不支持）。

**解决**：页面会显示"下载音频"按钮，下载后用本地播放器播放。

## 三、显示问题

### 3.1 图片打开显示乱码

**原因**（已修复）：早期版本没有图片查看器，图片被当作纯文本读取。

**解决**：更新到最新版本，图片会以查看器形式打开，支持翻页浏览。

### 3.2 缩略图不显示

**可能原因**：
- ffmpeg 未安装或不在 PATH 中
- 视频文件损坏或太短（不足 5 秒）
- 视频编码特殊，ffmpeg 无法提取帧

**排查**：
```bash
# 测试 ffmpeg 是否可用
ffmpeg -version

# 手动测试缩略图生成
ffmpeg -ss 5 -i "视频路径" -vframes 1 -vf scale=320:-1 -y test.jpg
```

缩略图失败会降级为显示文件图标，不影响播放。

### 3.3 文件名显示异常

**原因**：文件名含特殊字符（HTML 标签、引号等）被转义显示。

**这是安全措施**：防止 XSS 攻击，所有文件名在显示前会经过 HTML 转义。`<script>` 会显示为 `&lt;script&gt;`，这是正常的。

## 四、上传问题

### 4.1 上传大文件失败

**可能原因**：
- 文件超过 10GB 限制
- 网络不稳定导致连接断开
- 磁盘空间不足

**排查**：
- 检查文件大小是否超过 10GB
- 检查目标磁盘剩余空间
- 尝用更稳定的网络（有线比 WiFi 稳定）

### 4.2 上传速度很慢

**原因**：WiFi 带宽限制。局域网上传速度取决于路由器和 WiFi 标准。

**优化**：
- 使用 5GHz WiFi（比 2.4GHz 快）
- 手机靠近路由器
- 电脑用有线连接路由器

## 五、配置问题

### 5.1 忘记密码

**解决方式一**：管理后台修改（推荐）

登录后进入管理后台（`/admin`），在"修改密码"区域输入旧密码和新密码即可。

**解决方式二**：删除 `shares.json` 中的 `password` 字段，重启服务会自动生成新密码。

**解决方式三**：命令行指定密码：
```bash
python -m media_server --password 新密码
```

### 5.2 文件删除

浏览页每个文件右侧有删除按钮（红色垃圾桶图标），点击后确认即可删除。

> 注意：删除操作不可恢复，请谨慎操作。

### 5.2 共享目录添加后不显示

**可能原因**：
- 路径不存在或已移动
- 路径不是绝对路径（如 `./videos` 不行，必须 `D:\videos`）
- 没有访问权限

**排查**：
- 在管理后台确认路径正确
- 在文件管理器中确认路径存在
- 检查 `shares.json` 内容

### 5.3 想修改端口号

**方式一**：命令行参数
```bash
python -m media_server --port 9090
```

**方式二**：修改启动脚本或快捷方式，加上 `--port` 参数。

端口修改后需要重新添加防火墙规则：
```powershell
netsh advfirewall firewall add rule name="Media Server" dir=in action=allow protocol=tcp localport=9090
```

## 六、Windows 专属问题

### 6.1 控制台中文乱码

**原因**：Windows 控制台默认 GBK 编码，部分 Unicode 字符无法显示。

**影响**：仅显示问题，不影响功能。日志文件（`media_server.log`）是 UTF-8 编码，内容正常。

**解决**（可选）：
```powershell
# 切换控制台到 UTF-8
chcp 65001
```

### 6.2 Python 命令找不到

**报错**：`'python' 不是内部或外部命令`

**解决**：
- 安装 Python 时勾选 "Add Python to PATH"
- 或手动添加：`系统属性` -> `环境变量` -> `Path` 中添加 Python 安装目录
- 临时方案：使用完整路径 `C:\Users\你的用户名\AppData\Local\Programs\Python\Python313\python.exe`

## 快速检查清单

遇到问题时按顺序检查：

```
[ ] 手机和电脑连同一个 WiFi？
[ ] 服务器在运行？（控制台有日志）
[ ] 访问地址和端口正确？（看启动日志）
[ ] 防火墙规则已添加？
[ ] 网络配置文件是"专用"不是"公用"？
[ ] Python 和 ffmpeg 在 PATH 中？
```
